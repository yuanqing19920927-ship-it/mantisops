# MantisOps 版本路线图

> 最后更新：2026-03-27

---

## V1 — 基础监控平台（已完成）

核心监控能力搭建，覆盖服务器、数据库、端口、资产四大板块。

| 功能 | 说明 | 状态 |
|------|------|------|
| Agent 指标采集 | Go Agent 通过 gRPC 上报 CPU/内存/磁盘/网络/容器/GPU 指标，30 秒周期 | ✅ |
| WebSocket 实时推送 | 全局单连接，服务器指标实时更新到前端 | ✅ |
| VictoriaMetrics 存储 | 所有指标写入 VM，支持 PromQL 查询 | ✅ |
| 仪表盘 | 全局概览：服务器状态列表 + 端口摘要 + 资源排行 + 统计卡片 | ✅ |
| 服务器列表 | 卡片/表格双视图，实时 CPU/内存/磁盘进度条 | ✅ |
| 服务器详情 | 基本信息 + 实时概览（含 GPU）+ Docker 容器表格 + 运行业务 | ✅ |
| 端口监控 | TCP 端口探测规则 CRUD + 实时状态卡片 + 自动刷新 | ✅ |
| 本地业务管理 | 按服务器分组的资产表格，CRUD + 技术栈标签 | ✅ |
| 数据库 RDS 监控 | 阿里云 RDS 实例列表 + 实时指标瓦片 | ✅ |
| 阿里云 ECS 集成 | 通过 CloudMonitor API 采集 14 台 ECS 实例指标 | ✅ |
| Kinetic Observatory 设计系统 | 深色/浅色双主题，玻璃拟态 + 发光效果 + Material Symbols | ✅ |
| Nginx 统一入口 | :3080 反代 HTTP/WebSocket/VictoriaMetrics | ✅ |

## V2 — 历史数据 + 认证 + 告警（已完成）

增加数据回溯、安全访问控制和主动告警能力。

| 功能 | 说明 | 状态 |
|------|------|------|
| 历史趋势图表 | 服务器详情页：1h/6h/24h/7d 时间范围，CPU/负载/内存/磁盘/网络/GPU 共 9 个图表 | ✅ |
| 数据库历史趋势 | 数据库详情页：RDS 实例历史指标图表 | ✅ |
| JWT 登录鉴权 | 手写 HS256 JWT，7 天有效期，全局路由守卫 + Axios 拦截器 | ✅ |
| 服务器名称编辑 | 服务器详情页内联编辑 display_name | ✅ |
| 主题切换 | 顶栏太阳/月亮按钮，深色/浅色一键切换 | ✅ |
| 告警引擎 | 后端 30 秒轮询评估，连续 N 次确认触发/恢复，内存状态机 | ✅ |
| 10 种告警类型 | 服务器离线、端口 DOWN、CPU/内存/磁盘/容器/GPU 温度/GPU 显存/网络入站/出站 | ✅ |
| 告警事件管理 | 事件表格：级别/名称/目标/状态，手动确认（静默），自动恢复 | ✅ |
| 告警规则 UI | 前端 CRUD，动态生效，启用/禁用/删除自动处理 firing 事件 | ✅ |
| 通知渠道 | 钉钉机器人 + 通用 Webhook，异步发送 + 认领机制 + 失败重试 | ✅ |
| 通知投递追踪 | 每事件×每渠道独立记录，可查看投递状态和错误信息 | ✅ |
| 通知铃铛 | 顶栏红色徽章 + 下拉快速预览 + 新告警脉冲动画 | ✅ |
| 告警可靠性 | 事件+通知原子事务、新鲜度 TTL 防陈旧误报、目标消失自动清理 | ✅ |

NAS 设备接入与监控，兼容群晖和飞牛 OS。

**设计文档：** `docs/superpowers/specs/2026-03-29-nas-monitoring-design.md`

| 功能 | 说明 | 状态 |
|------|------|------|
| NAS 设备管理 | 设置页添加/编辑/删除 NAS 设备，SSH 连通性测试，NAS 类型自动检测 | ✅ |
| SSH 采集引擎 | 通过 SSH 命令定时采集 NAS 指标，支持群晖 DSM 7.x 和飞牛 fnOS | ✅ |
| RAID 监控 | /proc/mdstat 解析，支持 RAID 1/5/6，降级/重建检测 | ✅ |

| NAS 列表页 | 统计卡片 + 设备卡片（CPU/内存/RAID/磁盘/网络/UPS 实时展示） | ✅ |
| NAS 详情页 | 设备信息 + RAID 阵列 + 存储卷 + 硬盘健康表 + UPS + 历史趋势图表 | ✅ |
| NAS 告警 | 6 种类型：离线/RAID 降级/S.M.A.R.T. 异常/温度/卷空间/UPS 电池 | ✅ |
| WebSocket 实时推送 | nas_metrics + nas_status 消息，列表页实时更新 | ✅ |
| 群晖专属 | DSM 版本/型号/序列号采集，套件状态 | ✅ |
| 飞牛专属 | fnOS 版本检测（/etc/issue），Btrfs 健康检查 | ✅ |

## V4 — 远程运维 + 权限管理

### 4.1 多用户权限管理（已完成）

**设计文档：** `docs/superpowers/specs/2026-03-29-multi-user-rbac-design.md`

| 功能 | 说明 | 状态 |
|------|------|------|
| 多用户账号 | 管理员创建/编辑/禁用/删除用户，bcrypt 密码哈希 | ✅ |
| 三级角色 | `admin` / `operator` / `viewer`，RequireRole 中间件，路由三级分组 | ✅ |
| 资源级权限 | 按服务器分组/单服务器/数据库/探测规则配置可见范围，树形权限配置页 | ✅ |
| 即时踢下线 | JWT token_version 机制 + TokenVersionCache 内存缓存 | ✅ |
| 强制改密 | 管理员创建用户后首次登录强制改密，专属改密页面 | ✅ |
| REST 资源过滤 | 所有 List 接口按 PermissionSet 过滤，聚合接口 SQL 层过滤 | ✅ |
| WebSocket 权限过滤 | Hub 按连接权限过滤 metrics/alert/log 推送，admin-only 审计广播 | ✅ |
| 操作审计日志 | 用户管理操作全记录，PathSuffix 区分 reset-pwd/permissions | ✅ |
| 系统不变量 | 至少保留一个启用的 admin，禁止禁用/删除自己 | ✅ |
| 旧账号迁移 | 首次启动从 server.yaml 自动创建初始 admin | ✅ |
| 前端角色显隐 | 侧边栏/操作按钮/通知渠道 Tab 按角色显隐 | ✅ |
